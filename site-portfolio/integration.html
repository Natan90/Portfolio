Intégration dans l'équipe

Dès la première semaine, j'ai participé aux réunions hebdomadaires du lundi matin avec l'ensemble de l'équipe. Ces réunions m'ont permis de comprendre le fonctionnement de l'entreprise, d'écouter les projets en cours (marketing, contenu éditorial) et de prendre la parole sur l'avancement du Pass Wouafer's chaque semaine.

Au quotidien, je travaillais en open space dans les locaux de l'entreprise. J'étais le seul développeur sur le projet. Les échanges avec le reste de l'équipe relevaient du cadre professionnel et humain, pas d'une collaboration technique. Les points réguliers avec Sophie (direction) m'ont appris à adapter mon discours à un interlocuteur non-développeur : vulgariser les sujets techniques et hiérarchiser ce qui comptait pour l'entreprise.

Intégration dans le projet

Le projet m'a été confié avec une contrainte claire : travailler dans l'environnement déjà en place. L'entreprise utilisait WordPress comme CMS, Elementor comme page builder, et le serveur était déjà configuré avec une architecture de fichiers WordPress existante. Je n'avais pas le choix des outils : il fallait s'y adapter rapidement.

J'ai d'abord pris en main la base de code PHP existante avant d'y intégrer le plugin. Comprendre les conventions WordPress (hooks, filtres, structure de thème) et respecter l'architecture en place était un prérequis pour que mon code soit maintenable par l'équipe après mon départ.

Savoir-faire mobilisés

Travailler en équipe : participation aux réunions, points réguliers avec Sophie pour cadrer les priorités.

Rendre compte de son activité : chaque lundi, présentation orale de l'avancement structurée en fait / en cours / bloquant devant l'équipe.

S'approprier un environnement technique existant : prise en main de WordPress, Elementor et de l'architecture serveur imposée par l'entreprise, sans pouvoir repartir de zéro, avec l'obligation de comprendre l'existant avant de modifier quoi que ce soit.
